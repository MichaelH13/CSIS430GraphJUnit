Graph Class Part Deux

You should now have a functional Graph class (if not, you need to correct previous flaws).

For this assignment we will be adding some useful features to our class. First, you are to add a public static inner-class: 

public static class Edge<E> {
   public Edge(E source, E destination, int weight) {}
   public int getWeight() {}
   public E getSource() {}
   public E getDestination() {}
   public String toString() {}
}
You do not need to use this class as part of your internal implementation; it will be used to return paths as shown below.

Notes:

  • The "factory" method fromCSVFile() ought to construct and populate a graph from a Scanner with the assumption that the file has the following format: 
                      #vertices
                  vertex1
                  vertex2
                  ...
                  vertexN
                  #edges
                  source1,destination1,weight1
                  source2,destination2,weight2
                  ...
                  sourceN, destinationN, weightN
    Any sort of expected failure while processing the input stream should result in an IOException, including other exceptions from methods populating the graph.

  • shortestPathBetween should use Dijkstra's algorithm to find and return the shortest path between any two connected vertices. null is returned if no path exists.
