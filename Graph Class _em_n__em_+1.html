Graph Class Part n+1

You should still have a functional Graph class (if not, you need to correct previous flaws).

For this assignment we will be adding a couple more useful features to our class. First, if you have not already done so, make your Edge class Comparable: 

public class Edge implements Comparable {
   public int compareTo(Object o) {}
}

Now modify the graph class with three new methods: 

   public Edge<T> getEdge(T source, T destination) {}
   public List<Edge<T>> getEdges() {}
   public Graph<T> minimumSpanningTree() throws IllegalStateException {}

Notes:

  • getEdge() can return null if there is no such edge.
  • If the graph is undirected, then the list of edges returned by getEdges() should only include one edge for each pair of connected vertices; otherwise both edges should be included.
  • Your implementtion of minimumSpanningTree() can use either Prim's or Kruskal's algorithm (Javadoc must clearly identify which you've used). null should be returned if no spanning tree exists. Note that it is not legal to ask for the minimum spanning tree of a directed graph.
